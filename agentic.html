The SAP Context
SAP 클라우드 전환이
테스팅을 바꾸는 이유

클라우드로의 전환은 단순한 IT 업그레이드가 아닌 운영 모델의 근본적 전환이다. 에버그린 시스템 시대에 분기·월·주 단위 업데이트를 따라가는 지속적 품질 보증이 필수가 됐다.

🏗️

레거시 → 클라우드: 거대한 전환

전체 비즈니스 프로세스 마이그레이션, 데이터 검증, 통합 재설계. SAP 고객이 직접 밝힌 수치: 테스팅만으로도 전환 프로젝트 전체 시간·노력의 50%를 차지한다.

🔄

클라우드 이후: 에버그린 시대

클라우드에 도착한 이후가 진짜 도전의 시작. 분기·월·주 단위로 쏟아지는 SAP 업데이트를 따라가려면 지속적으로 테스트하고 검증할 수 있는 자동화 기반이 필수다.

🤖

Gartner 예측: AI 가치의 실체

2027년까지 AI 기반 ERP 가치의 30% 미만만이 생성형 AI에서 나올 것. 대부분의 가치는 임베디드 AI + Agentic AI에서 창출된다. 하지만 조직의 70%는 AI-ready 데이터를 갖추지 못할 것으로 예상.

🔗

SAP 통합 툴체인

Rise with SAP 방법론 + Tricentis AI. 기존 방식 대비 비용 30% 절감, 가치 실현 시간 35% 단축. Tricentis는 이 통합 툴체인의 핵심 QA 파트너로 포지셔닝.

The AI Journey
Tricentis AI 전략:
3단계 여정

AI 도입은 한 번에 이루어지지 않는다. Tricentis가 제시하는 3단계 여정을 통해 팀의 준비 수준에 맞게 단계적으로 진화한다.

단계 1
Bring Your Own AI
MCP(Model Context Protocol)를 통해 개발자가 이미 사용 중인 Cursor, VS Code 등 환경 안으로 Tricentis 역량을 가져온다. 개발자는 코딩만 하는 것처럼 느끼지만, 자동으로 QTest 업데이트 + Tosca 자동화 + 커버리지 분석이 이루어진다.
단계 2
AI-Assisted
AI 추천, 자동 생성, Self-healing 기능이 팀 생산성을 증폭시킨다. 테스트 실행 우선순위 자동화, Quality Intelligence 접목, 결과 학습을 통해 고부가가치 문제 해결에 집중할 수 있게 된다.
단계 3
Agentic AI
완전 자율 에이전트 오케스트레이션. AI Workspace에서 여러 전문 Agent가 협업하며 테스트 생성, 실행, 분석, 결함 보고를 자동으로 처리한다. 사람은 거버넌스와 예외 처리에 집중한다.

Tricentis AI Workspace
라이브 데모 주요 내용

이 웨비나의 핵심은 AI Workspace 라이브 데모. Agent Builder, 워크플로우 오케스트레이션, MCP 연동을 직접 시연했다.

🏭

AI Workspace 구조

  • AI 트랙 채팅 (자연어 협업 환경)
  • AI 워크플로우 관리 (다중 Agent 오케스트레이션)
  • Agent Builder (전문 Agent 제작)
  • Knowledge Base (기업 표준·정책 적용)
  • MCP Mesh (외부 Agent·도구 연결 허브)
🤖

Agent 설계 원칙

  • "슈퍼 Agent 금지" — 한두 가지를 정말 잘하는 전문 Agent
  • 각 Agent에 목적·스킬셋 명확히 정의
  • Agent ↔ Agent 간 소통·협업 가능
  • 프롬프트 엔지니어링이 새로운 QE 핵심 역량
  • AI는 실수함 → 범위 제한으로 정확도 확보
🔌

MCP (Model Context Protocol)

Tricentis qTest·Tosca·Sea Lights용 MCP 서버 출시. 개발자 환경(Cursor, VS Code 등)에서 자연어로 Tricentis 도구를 직접 조작 가능. 코드 변경 시 자동으로 테스트 자산 업데이트. 고객·파트너 MCP도 Workspace에 연결 가능.

⚙️

데모에서 보여준 워크플로우

  • 요구사항 분석 Agent 실행
  • 수동 테스트 자동 생성 + 요구사항 연결
  • Agentic 테스트 자동화로 자동화 테스트 생성
  • 한 번의 클릭으로 전체 파이프라인 실행
  • Agent 상태·진행 현황 실시간 모니터링

Jarvis Analysis
PerfecTwin 관점
시사점 & 전략적 함의

이 웨비나가 PerfecTwin ERP Edition 전략에 미치는 영향 분석.

  • 🎯 이 웨비나의 성격: 제품 데모 포함 기술 전략 발표. Tricentis가 SAP 고객에게 "AI Workspace + MCP"를 통해 Agentic 테스팅을 실현하는 구체적 로드맵을 제시한 가장 상세한 콘텐츠. Lab Preview 기능들이 포함되어 있어 실제 제품 출시 전 고객 반응 검토 목적도 있다.
  • ⚠️ 가장 위협적인 포인트 — "개발자 환경 침투": MCP를 통해 Cursor·VS Code 같은 개발자 도구 안에 QA를 자연스럽게 내장하는 전략은 매우 영리하다. "개발자는 코딩만 하는 것처럼 느끼지만 QA는 자동으로"라는 가치 제안은 경쟁사가 쉽게 복제하기 어렵다. PerfecTwin은 현재 이 영역에 대응 전략이 없다.
  • 💡 PerfecTwin 차별화 포인트: Tricentis AI는 새 코드 생성 단계에서 AI를 통합하는 방향. PerfecTwin은 이미 운영 중인 SAP ERP의 실거래 기반 회귀 테스트에 집중. "클라우드에 도착한 이후 에버그린 업데이트 대응" 시나리오에서 PerfecTwin의 실거래 캡처→재현 가치가 Tricentis의 접근과 상호보완적으로 포지셔닝 가능하다.
  • 📌 "슈퍼 Agent 금지" 원칙은 시장 교육: Brad Pursell이 강조한 "전문 Agent 설계" 원칙은 고객들이 Agent 도입 시 범하기 쉬운 실수를 예방하는 시장 교육이다. PerfecTwin도 영업 시 "테스팅 전문 Agent"로서의 역할을 명확히 정의하고 커뮤니케이션할 필요가 있다.
  • 📅 주목 시점: AI Workspace가 Lab Preview에서 GA(일반 출시)로 전환되는 시점이 경쟁 환경의 분수령. 2025년 하반기~2026년 상반기로 예상된다. 해당 출시 전에 PerfecTwin의 "SAP 전문 테스팅 Agent" 포지셔닝을 강화해야 한다.
